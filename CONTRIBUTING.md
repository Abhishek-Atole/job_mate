# Contributing

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Install dependencies (see README.md)
4. Make your changes
5. Run the test suite: `bash test_all.sh`
6. Commit and push

## Code Standards

- Follow existing code style (no comments unless necessary)
- Keep functions small and focused
- Add API tests for new endpoints in `test_all.sh`
- Run all tests before submitting

## Pull Request Process

1. Ensure all 88+ tests pass
2. Update README.md if adding new features
3. PRs require one approval
